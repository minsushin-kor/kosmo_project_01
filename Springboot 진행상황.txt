================================================================================
Spring Boot 개발 프로젝트 통합 진행상황 및 세부 기능 명세
================================================================================

본 문서는 중고차 거래 플랫폼의 Spring Boot 백엔드 애플리케이션에 구현된 모든 세부 기능과 도메인별 설계 사양을 상세히 기록한 통합 진행상황 보고서입니다.

--------------------------------------------------------------------------------
1. 데이터베이스 모델링 및 JPA 엔티티 설계 (Entity & Repository)
--------------------------------------------------------------------------------
* 엔티티 클래스 정의 및 DB 매핑 (@Entity, @Table, @Id, @Column 사용)
  - Member (일반 회원): 이메일, 패스워드, 이름, 연락처, 프로필 이미지, 역할(MEMBER, ADMIN) 관리.
  - Company (상사): 사업자번호, 상사명, 마스터 이메일, 패스워드, 주소, 연락처, 로고 이미지, 멤버십 가입 여부, 상사 등급(NORMAL, TOP_5, CARE_REQUIRED), 이탈위험점수(risk_score), 골든 뱃지 상태(golden_badge_status) 관리.
  - Dealer (상사 소속 딜러): 소속 상사 연계, 로그인 ID, 패스워드, 이름, 연락처, 프로필 이미지, 활동 상태(ACTIVE, WITHDRAWN), 딜러 등급(NORMAL, CARE_REQUIRED), 이탈위험점수(risk_score) 관리.
  - Car (중고차 매물): 소유주(Member 혹은 Dealer) 관계 연계(1:1 독점 소유 제약 검증), 연식, 제조사, 모델, 옵션, 외관상태, 변속기, 번호판 주/지역, 상태 점수, 누적 주행거리, 외판 색상, 내장 색상, MMR(도매가), 판매 희망 가격, 차량 상태(REGISTERED, AUCTION_ACTIVE, SOLD, WITHDRAWN) 관리.
  - CarImage (차량 다중 사진): 차량 연계(1:N), 이미지 저장 경로 URL, 대표 대표 사진 여부(is_main) 관리.
  - Auction (블라인드 경매 세션): 차량 연계(1:1), 경매 시작 일시, 경매 종료 일시, 경매 진행 상태(ACTIVE, COMPLETED, CANCELLED) 관리.
  - Bid (경매 입찰 기록): 경매 세션 연계, 입찰 딜러 연계, 입찰 금액 관리.
  - Transaction (최종 거래 및 정산 이력): 차량 연계, 구매자 유형(MEMBER/DEALER) 및 ID, 판매자 유형(MEMBER/DEALER) 및 ID, 체결 가격, 수수료율, 최종 수수료 금액 관리.
  - MembershipHistory (상사 멤버십 가입 이력): 상사 연계, 가입 결제 금액, 멤버십 시작일, 종료일, 멤버십 상태(ACTIVE, EXPIRED) 관리.
  - Wishlist (관심 매물/찜하기): 일반 회원 또는 딜러와 차량 매물 간의 관심 등록 매핑 관리.
  - Coupon (수수료 감면 및 멤버십 할인 쿠폰): 쿠폰명, 쿠폰 유형(COMMISSION_DISCOUNT, MEMBERSHIP_DISCOUNT), 수수료 감면율/할인율, 딜러/상사 연계, 쿠폰 사용 상태(UNUSED, USED, EXPIRED), 발급일, 만료일 관리.

* JPA 연관관계 및 제약 조건 제어
  - 영속성 전이(CascadeType.ALL) 및 고아 객체 제거(orphanRemoval=true)를 활용하여 Car와 CarImage 간의 생명주기를 동기화.
  - DB Unique 제약 조건과 인덱스를 설정해 한 딜러가 동일 경매에 중복 입찰하는 것을 원천 차단(uq_auction_dealer).
  - 회원의 중복 찜하기 방지를 위한 조건부 유니크 인덱스 적용.

--------------------------------------------------------------------------------
2. 보안 인프라 및 인증/인가 (Security & JWT Token)
--------------------------------------------------------------------------------
* JWT 기반 인증 시스템 구축
  - SecurityConfig 설정을 통해 각 역할(MEMBER, DEALER, COMPANY, ADMIN)에 따른 API 접근 제어 규칙 바인딩.
  - JwtProvider 컴포넌트 개발: JWT 토큰 발급, 만료일 설정, Claims 추출 및 사용자 검증 로직 구현.
  - JwtAuthenticationFilter를 통해 Request Header의 Bearer 토큰을 분석해 스프링 시큐리티 컨텍스트에 인증 객체 저장.
* 다중 역할(Multi-Role) 로그인 처리
  - 일반 회원/상사 마스터: 이메일 기반 로그인 검증 처리.
  - 상사 소속 딜러: 상사 마스터가 부여한 login_id 기반 로그인 검증 처리.
  - BCryptPasswordEncoder 빈 등록을 통해 패스워드 단방향 암호화 및 검증 통일.

--------------------------------------------------------------------------------
3. 회원가입 및 사용자 관리 API
--------------------------------------------------------------------------------
* 일반 회원 회원가입 (`POST /api/auth/signup/member`)
  - 프로필 이미지 URL 단건 등록 기능 포함.
  - 가입 시 보유 차량 정보(보유 여부, 보유 차량 사진 URL, 제조사, 모델, 주행거리, 연식)를 포함하여 등록하는 회원 특화 사양 반영.
* 상사 마스터 회원가입 (`POST /api/auth/signup/company`)
  - 사업자 등록 번호 중복 검증 로직 구현.
  - 상사 로고/프로필 이미지 URL 단건 등록 기능 포함.
* 통합 로그인 API (`POST /api/auth/login`)
  - 이메일 또는 login_id를 분기하여 회원의 역할에 알맞은 토큰 발급 및 사용자 프로필 요약 반환.
* 딜러 계정 관리 API (상사 마스터 전용 권한 필요)
  - 딜러 등록 (`POST /api/company/dealers`): 마스터가 소속 딜러의 아이디, 패스석, 이름, 연락처, 프로필 이미지를 직접 지정하여 가입 처리 (비밀번호 BCrypt 암호화 저장).
  - 딜러 명단 제외 (`DELETE /api/company/dealers/{dealerId}`): 딜러 상태(Status) 필드를 `WITHDRAWN`으로 안전하게 비활성화 처리.

--------------------------------------------------------------------------------
4. 매물(중고차) 등록 및 조회 서비스 API
--------------------------------------------------------------------------------
* 차량 매물 등록 (`POST /api/cars`)
  - 일반 회원 또는 딜러 권한에 따라 소유자 매핑 검증 처리.
  - 한 매물에는 반드시 일반 회원 1명 혹은 딜러 1명만 소유주로 바인딩될 수 있도록 비즈니스 체크 제약(chk_cars_owner) 적용.
  - 다중 이미지 일괄 등록 및 대표 이미지(`is_main=true`) 지정 기능 제공.
* 차량 단건 상세 조회 (`GET /api/cars/{carId}`)
  - 차량의 모든 상세 스펙, 이미지 목록, 소유주의 상세 프로필 정보, 그리고 골든 뱃지 혜택 적용 유무를 단일 조회 응답으로 조립하여 반환.
* 차량 다중 필터 검색 및 페이징 조회 (`GET /api/cars`)
  - 제조사, 모델, 연식, 변속기, 외판 색상, 상태 점수, 가격대 등 동적 검색 필터 지원 (Specification 사용).
  - 페이징 처리를 통한 부하 분산 및 정렬 기준 제공.

--------------------------------------------------------------------------------
5. 경매(Auction) 및 입찰(Bid) API
--------------------------------------------------------------------------------
* 경매 세션 개설 및 검증
  - 일반 회원이 차량 등록 시 경매 자동 생성 (시작일시 및 종료일시 입력 필수).
  - 경매 시작 시간이 현재 시간 기준 5분 이전이 아닌지 검사하여 네트워크 지연 등 보정.
  - 경매 종료 시간은 시작 시간보다 미래여야 하며, 경매 최장 유지 기간은 3일(72시간)을 초과하지 못하도록 검증.
* 딜러 블라인드 입찰 (`POST /api/auctions/{auctionId}/bids`)
  - 딜러만 입찰에 참여할 수 있도록 권한 검증.
  - 이미 입찰에 참여한 딜러가 중복 입찰하는 행위를 예외 처리로 완벽 방지.
* 입찰 현황 조회 (`GET /api/cars/{carId}/bids`)
  - 차량의 소유주(일반 회원)만 본인 차량의 실시간 블라인드 입찰 리스트 및 딜러 정보를 볼 수 있도록 인가 검증 포함.

--------------------------------------------------------------------------------
6. 경매 마감, 거래 체결 및 평점 API
--------------------------------------------------------------------------------
* 경매 자동/수동 마감 및 낙찰 처리
  - 경매 종료 후 가장 높은 입찰가를 부른 딜러를 낙찰자로 자동 선정.
  - Transaction(거래 정보) 데이터 생성 및 차량 상태를 `SOLD`로 업데이트.
* 딜러 차량 즉시 구매 API (`POST /api/cars/{carId}/purchase`)
  - 일반 회원이 딜러 소유 매물을 즉시 구매 신청할 때 거래 체결 처리.
  - 본인 차량을 본인이 직접 구매하는 행위 방지 예외 처리.
* 거래 리뷰 및 평점 등록 (`POST /api/transactions/{transactionId}/reviews`)
  - 거래 완료 건에 대해 상대방 딜러에 대한 평점(별점) 및 한줄평 등록 지원.

--------------------------------------------------------------------------------
7. 관심 차량 및 마이페이지 API
--------------------------------------------------------------------------------
* 관심 차량(찜하기) 토글 (`POST /api/wishlists/{carId}`)
  - 일반 회원 또는 딜러가 관심 매물로 등록하거나 해제 처리 (중복 방지 유니크 제약 적용).
* 역할별 통합 마이페이지 API (`GET /api/users/me/profile`)
  - 로그인 세션 기반으로 프로필 데이터 응답.
  - 일반 회원: 등록 매물, 낙찰/구매 거래 내역, 찜한 차량 목록.
  - 딜러: 입찰 참여 내역, 성사 거래 내역, 찜한 차량 목록.
  - 상사 마스터: 소속 딜러 명단, 상사 정보, 소속 딜러들의 통합 거래 건수 등.

8. 이탈 방지 딜러 수수료 50% 감면 쿠폰 시스템
--------------------------------------------------------------------------------
* 감면 쿠폰 발급 제약
  - 이탈 위험도 점수(Risk Score)가 75점 이상인 딜러에게 수수료 50% 감면 쿠폰(기본 수수료 0.3% -> 0.15% 감면 적용) 자동 발행.
  - 한 번 쿠폰을 발급받은 딜러는 중복 지급 방지를 위해 최소 한 달(30일)이 경과해야 재발급이 가능하도록 최근 발급 일자 안전 검증 필터 구현.
* 거래 시 쿠폰 선택 적용 및 차감
  - 딜러가 낙찰 건 정산 또는 매물 구매 완료 시 본인이 소유한 쿠폰을 직접 선택하여 수수료 차감 적용.
  - 최종 수수료 금액(`commission_amount`)을 50% 할인액으로 반영하고 해당 쿠폰 상태를 `USED`로 갱신.

--------------------------------------------------------------------------------
9. 골든 뱃지 및 최우수 상사 혜택 쿠폰 시스템
--------------------------------------------------------------------------------
* 골든 뱃지 자격 산정 및 갱신 (`updateCompanyTiersAndBadges()`)
  - 매일 자정 전체 상사 소속 딜러들의 성사 거래량을 합산 정렬.
  - 실적 기준 상위 5% 최우수 상사 식별 후 등급을 `"TOP_5"`로 지정하고 골든 뱃지 상태(`golden_badge_status = true`) 활성화.
  - 상위 5% 최우수 상사에 유료 멤버십 가입 시 사용할 수 있는 20% 할인 쿠폰을 자동 발급 (미사용 상태의 중복 발급 방지 적용).
  - 상위 5%에 들지 못한 일반 상사는 강등(`"NORMAL"`) 및 골든 뱃지 박탈 처리.
  - 단, 이탈 위험 점수가 높게 나와 특별 관리가 필요한 상사(`"CARE_REQUIRED"`) 등급이 강제 초기화되는 것을 막기 위해 예외 보호 조건 추가 적용.

10. FastAPI 연동 및 AI 차량 추천 API
--------------------------------------------------------------------------------
* FastAPI HTTP REST 클라이언트 연동 ([AiClient.java])
  - `RestTemplate`을 사용한 통신 인프라 설계 (연결/읽기 타임아웃 5초 설정으로 병목 방지).
  - Jackson 라이브러리의 `@JsonProperty`를 활용해 FastAPI 데이터 특징 모델(대소문자 및 언더바 조합)과의 자동 바인딩 제공.
  - 모든 예외 정보 및 진행 안내 로깅 메시지를 직관적인 한글로 설계.
* AI 맞춤 차량 추천 API 연동 (`GET /api/dealers/recommendations`)
  - 딜러가 추천 API 호출 시 `AiClient`를 통해 FastAPI 서버의 `/api/ai/recommendations`에 추천 요청 전달.
  - FastAPI로부터 추천받은 차량 ID 목록을 받아 DB의 매물 정보 목록으로 조립하여 상세 제공.
  - AI 서버 미구동 혹은 추천 목록이 비어있을 때 빈 화면 노출을 막기 위해 최신 매물 5개를 자동으로 채워주는 안전한 Fallback 로직 반영.

11. 이탈 방지 예측 배치 자정 스케줄러 (Midnight Batch Scheduler)
--------------------------------------------------------------------------------
* 자정 스케줄러 실행 및 딜러/상사 위험 등급 업데이트
  - `@EnableScheduling` 및 `@Scheduled(cron = "0 0 0 * * ?")` 기반으로 동작.
  - 매일 자정 전체 활성 딜러들의 실제 활동 수치 집계:
    * 마지막 활동 경과일: 최근 입찰/거래 시각과 현재 시간의 차이 연산 (기록 없을 시 180일로 지정)
    * 최근 60일 거래 건수 및 이전 거래 건수
    * 사이트 이용률: (딜러 입찰수 / 전체 경매수) 비율 연산 (최소 0.0 ~ 최대 1.0 범위 보정)
    * 평균 거래 단가 계산 (기록 없을 시 1,300만 원 기본값 부여)
  - 추출한 특징 데이터를 FastAPI 딜러 이탈 예측 API로 전송해 이탈 확률을 예측받아 백분율 환산 점수(`risk_score = probability * 100`)로 DB 업데이트.
  - 위험 점수가 75점 이상인 딜러는 `"CARE_REQUIRED"` 등급 부여.
  - 상사는 소속 딜러들의 집계 데이터를 기반으로 FastAPI 상사 이탈 예측 API를 호출해 `risk_score`를 저장하고 70점 이상인 상사는 `"CARE_REQUIRED"` 등급 부여.
  - 등급 갱신 종료 직후, 이탈 딜러 수수료 할인 쿠폰 발급(`issueRiskCoupons()`) 및 골든 뱃지 갱신(`updateCompanyTiersAndBadges()`) 배치를 순차 구동시켜 전체 관리 스케줄 자동 연쇄 구동.
* 관리자 전용 배치 강제 트리거 API (`POST /api/admin/ai/churn-batch`)
  - 자정까지 대기할 필요 없이, 관리자 권한을 가진 로그인 사용자가 배치를 수동으로 즉시 구동할 수 있도록 테스트용 트리거 엔드포인트 제공.
