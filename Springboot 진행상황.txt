**엔티티 설계**: `Member`, `Company`, `Dealer`, `Car`, `CarImage`, `Auction`, `Bid`, `Transaction`, `MembershipHistory` 클래스 작성 (`@Entity`, `@Table` 지정).

**연관관계 매핑**:
  - `Company` (1) : `Dealer` (N) 단방향/양방향 매핑.
  - `Car` (1) : `CarImage` (N) 매핑 (CascadeType.ALL, orphanRemoval=true 적용).
  - `Auction` (1) : `Bid` (N) 매핑.

**JPA Repository 정의**: 각 엔티티에 대응하는 `JpaRepository` 인터페이스 선언.

**기본 매핑 테스트**: JUnit을 사용해 샘플 데이터의 INSERT 및 SELECT 동작 여부 테스트 검증.

**Security Config 설정**: 특정 API 엔드포인트 권한 허용 규칙 작성.

**JWT Provider 구현**: 토큰 생성, 검증, 파싱(Claims 추출) 컴포넌트 개발.

**다중 역할 UserDetailsService 구현**:
  - 이메일을 통한 일반 회원 및 상사 마스터 로그인 처리.
  - 상사에서 부여한 `login_id`를 통한 딜러 로그인 처리.

**회원가입/로그인 API (보유 차량 정보 기능 반영 완료)**:
  - 일반 회원 회원가입 (`POST /api/auth/signup/member` - 프로필 이미지 URL 1개 및 보유 차량 정보 [보유 여부, 보유 차량 사진 URL, 제조사, 모델, 주행거리, 연식] 추가 등록 기능 포함)
  - 상사 마스터 회원가입 (`POST /api/auth/signup/company` - 프로필 이미지 URL 1개 추가 등록 기능 포함)
  - 통합 로그인 및 토큰 반환 (`POST /api/auth/login`)

**딜러 계정 생성 및 해지 (상사 마스터 전용 - 프로필 이미지 기능 반영 완료)**:
  - 딜러 등록 (`POST /api/company/dealers` - 비밀번호 BCrypt 암호화, 프로필 이미지 URL 1개 추가 등록 기능 포함)
  - 딜러 명단 제외 (`DELETE /api/company/dealers/{dealerId}` - Status를 `WITHDRAWN`으로 비활성화)

**차량 등록 및 조회 서비스**:
  - 회원/딜러 차량 등록 (`POST /api/cars`)
  - 다중 이미지 첨부 및 대표 이미지(`is_main=true`) 지정 로직.
  - 다중 필터 검색 및 페이징이 포함된 차량 목록 조회 (`GET /api/cars`).

**경매 생성 및 블라인드 입찰 (경매 시간/기간 설정 고도화 완료)**:
  - 일반 회원 차량 등록 시 경매 생성 서비스 연계 (시작일시는 일반 회원이 정한 날짜와 시간부터로 동작하며, 종료일시는 시작일시 기준 최대 3일(72시간)을 넘지 않도록 검증).
  - 딜러 입찰 등록 (`POST /api/auctions/{auctionId}/bids`).
  - **입찰 제약 검증**: 서비스 계층에서 해당 딜러가 이미 입찰했는지 여부를 검증하고, DB Unique 제약 조건(`uq_auction_dealer`)을 활용해 한 딜러가 2번 이상 입찰하는 행위를 완전 원천 차단.
  - 판매자 전용 내 차량의 실시간 입찰 현황 리스트 조회 (`GET /api/cars/{carId}/bids`).

**경매 종료 및 낙찰**:
  - 경매 마감 처리 메서드 작성.
  - 최고 입찰 가격을 제시한 딜러 추출 후 `Transaction` 생성 및 해당 차량 상태 `SOLD` 처리.
  - 낙찰 거래 완료 후 딜러 리뷰 및 별점 평가 등록 (`POST /api/transactions/{transactionId}/reviews`).

**관심 차량(찜하기) 토글 서비스**:
  - 관심 차량 등록/해제 토글 (`POST /api/wishlists/{carId}`).

**역할별 마이페이지 조회 API**:
  - 로그인 유저 프로필 조회 (`GET /api/users/me/profile` - 회원/딜러/마스터 공통분기 처리)
  - 본인 소속에 따른 등록 매물, 입찰 참여 내역, 성사 거래 내역 및 관심 등록(찜) 차량 목록 조회.

**일반 회원의 딜러 차량 즉시 구매 서비스**:
  - 일반 회원이 딜러 매물을 즉시 구매 신청 (`POST /api/cars/{carId}/purchase`).
  - 거래 내역(`Transaction`) 생성 (구매자: `MEMBER`, 판매자: `DEALER`) 및 차량 상태 `SOLD`로 업데이트.
  - 본인 매물을 본인이 직접 구매하는 행위 방지 예외 처리.

**차량 단건 상세 조회 API**:
    - 특정 매물의 전체 상세 스펙, 첨부 이미지 리스트, 소유자 정보(딜러 또는 일반 회원 프로필) 조회 (`GET /api/cars/{carId}`).

**딜러 수수료 감면 쿠폰 발급 및 적용 (이탈점수 75점 이상 딜러)**:
  - **할인 쿠폰 발급**: 예측된 이탈 위험도 점수(Risk Score)가 75점 이상인 모든 딜러에게 수수료 50% 감면 쿠폰(0.3% -> 0.15%)을 자동 발행 및 이력 저장.
  - **수수료 적용**: 딜러의 2가지 거래 체결 시점(① 일반 회원 차량 낙찰 시, ② 딜러 차량의 일반 회원 즉시 구매 완료 시) 수수료 산정 시, 딜러가 보유한 감면 쿠폰을 적용하여 `Transaction` 테이블에 할인된 수수료(0.15%)를 기록하고 쿠폰을 사용 완료 처리.

**골든 뱃지 부여 & 멤버십 할인 쿠폰 발급 (상위 5% 상사)**:
    - **골든 뱃지 부여**: 상위 5% 최우수 상사(`Company`)에 골든 뱃지(`golden_badge_status`) 상태를 지정하고, 상사/딜러 프로필 및 등록 매물 반환 DTO에 골든 뱃지 활성화 상태를 포함시켜 화면에 뱃지 아이콘이 노출되도록 연계.
    - **할인 쿠폰 발급**: 상위 5% 상사 등급 부여 시 해당 상사에 유료 멤버십 가입 시 사용할 수 있는 할인 쿠폰 정보 발행 및 이력 저장.
